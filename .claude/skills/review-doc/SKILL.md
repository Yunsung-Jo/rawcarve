# 문서 검토

## 목표

**작성자가 자기 글에서 못 보는 정밀도 결함·누락을, 의도를 모르는 검증자의 관점에서 잡는다.**

작성 직후의 저자는 텍스트가 아니라 자기 의도를 읽는다. 그래서 작성 스킬(`write-spec`·
`write-adr`·`write-report`·`write-investigation`·`write-reference`)의 자가점검을 통과한 뒤에도
**틀린 수치, 출처 혼재, 단정과 추론의 혼동, 깨진 참조·asset, 미실행 검증**이 남는다.

이 스킬의 포지셔닝(작성 자가점검과 중복이 아니다):

- **수치 규율**(원자료 재계산·출처 명기·단위 일치)은 `write-report` 자가점검에만 있다. 이를
  **나머지 4개 유형(spec·ADR·investigation·reference)으로 일반화**한다.
- **코드 참조 grep · 링크/asset 열기 · 주장한 검증의 실제 실행 · 유형별 능동 검증**은
  **어느 write-\* 자가점검에도 없다.** 이것이 순수 신규 가치다.

핵심 구분: 작성 자가점검은 "갖췄나"의 **수동 체크**다. 이 스킬은 "직접 다시 해봤나"의
**능동 검증**이다.

## 언제 사용하나

- `write-*`로 문서를 작성·수정한 **직후** (자가점검과 별개로 한 번 더)
- 커밋·머지(**리파인 패스 — main `--squash` 직전**) **전**, 여러 문서를 함께 점검할 때
- **적용 범위:** 수치·코드 참조·검증 결과·링크를 건드린 변경. 오타·표현 한 줄 수정에 7원칙
  풀패스는 과하다.

## 핵심 원칙

원칙 **1–4는 직접 재실행**한다(재계산·grep·열기·검증 실행). **5–7은 의도를 모르는 검증자의
눈으로 재독**하되, 필요하면 재실행한다. 어느 쪽이든 "확인했다"가 아니라 **"해봤다"**여야 한다.

### 1. 수치는 원자료에서 재계산한다 (본문 + 그림 캡션)

본문뿐 아니라 **그림 캡션·몽타주 라벨**(예: `tools/montage.py` 스펙의 `sub` "회색 0.918 →
0.002")의 수치까지, 원자료(`report.csv`·로그·파일 크기·타임스탬프)에서 **다시 산출**해 대조한다.
곱·합·비율·배수는 특히 자주 틀린다. 규모가 크면 스크립트로 독립 재산출한다.

- 예: "5,632 × 52 ≈ 22만"으로 적었으나 곱은 약 29만이고 원자료 실측은 221,961회였다. 어림식이
  실측과 어긋나면 그 이유(조기 종료 등)를 적는다.

### 2. 코드 참조는 grep으로 확인한다

함수명·라인 번호·시그니처·상수를 코드에서 직접 찾아 대조한다. 경로는 **패키지 접두까지** 정확히.

- 예: `carver/jpegdecode.py:172`가 현재도 그 함수의 시작 줄인지 grep으로 확인한다.

### 3. 링크와 asset은 대상을 연다

상호 참조(ADR↔spec↔investigation)·README 목록·reference 링크의 대상 파일과 앵커, 그리고
**이미지/asset 경로**(`docs/reports/assets/*.webp` 등)가 실제 존재하는지 연다. 작성 중 건
"아직 없는 미래 문서/asset" 링크가 깨진 채 남기 쉽다.

### 4. 주장한 검증을 실제로 실행한다

"무손실·회귀 없음·통과·동일"은 해당 검증(테스트·빌드·diff·재계산)을 **실제로 돌려** 뒷받침한다.
안 돌렸으면 **지금 돌리고** 결과를 본문에 기록한다.

- 예: 무손실 변경이라면서 기존 테스트를 안 돌렸으면, 지금 실행해 "54개 통과"를 확인·기록한다.
- **investigation 예외:** 랩노트는 시점 스냅샷이라 사후 검증 결과를 본문에 새로 삽입하지 않는다.
  미실행 주장은 "추론"으로 약화하고, 계산 오류·오타만 정정한다(→ 원칙 7·절차 4).

### 5. 단정과 추론을 구분한다

**직접 측정·실행으로 확인한 것만 단정한다.** 정황으로 판단한 것은 "추론"으로 표시하고 근거가
된 정황을 남긴다.

- 예: 메모리 대역폭을 계측하지 않고 "대역폭 포화"로 단정하지 말 것. → "진동 소멸 + 실전 배수
  > 단일 배수로 추론"처럼 근거와 함께 추론임을 명시한다.

### 6. 출처·단위·용어가 일관되어야 한다

한 수치가 문서 전체에서 같은 출처·기준에서 나왔는지, 측정 방식이 다르면(계측 오버헤드 포함 vs
순수 실행, budget 조건 차이) 섞이지 않고 각 수치에 출처가 붙는지 확인한다. 같은 개념의
**용어 표기**도 일관되는지("회색/gray/잔존" 혼용 등) 본다.

- 예: cProfile 193초(오버헤드 포함)와 순수 187.7초가 구분 없이 섞이면 독자가 막힌다.

### 7. 유형별 능동 검증 (재독이 아니라 행동으로)

해당 유형의 자가점검을 다시 보는 데 그치지 말고, 유형별 핵심 결함을 **능동으로** 확인한다:

- **spec:** "사용하는 모듈" 목록이 실제 `import`와 일치하는지 grep; 기술된 동작이 현재 코드와
  어긋나지 않는지(drift) 대조 — 어긋난 spec은 없느니만 못하다.
- **report:** 비기술 독자 기준으로 미정의 약어·내부 식별자를 훑고; **민감정보(개인 사진 내용)
  노출·블러 적용**을 확인.
- **investigation:** "기각된 가설" 절이 비지 않았는지; 각 단계가 5요소(가설/실험/예측/증거/판단)를
  갖췄고 예측이 증거보다 먼저 적혔는지; **몽타주·이미지의 시각적 내용 묘사가 없는지**(지표·결함
  패턴만 — 있으면 삭제); 스냅샷 불변(코드 drift로 갱신하지 않음) 준수.
- **ADR:** 각 대안에 정량 기각 이유가 있는지; 대체 시 기존 ADR 상태가 `Superseded`로 바뀌었는지.
- **reference:** 인용한 포맷 사실(매직바이트·청크 구조·RFC 조항)이 출처와 맞는지 표본 대조.
- **공통 문체:** [../shared/doc-style.md](../shared/doc-style.md) 위반(의인화·감정어·과정 포장
  제목 '규명'·'~에서 ~로')이 없는지.

## 절차

1. **대상·유형 식별** — `git diff`로 이번에 건드린 문서를 추린다(경로:
   `docs/{specs,adr,reports,investigations,reference}`). 수치·참조·검증을 건드린 변경에 한정.
2. **원칙 1–7 수행** — 재계산·grep·열기·검증 실행을 실제로 한다.
3. **발견 → 조치 정리** — 검토 중 휘발성 작업 메모로만 쓴다(파일로 저장하지 않는다).
4. **수정 + 전파** — 바뀐 수치를 인용하는 다른 문서·README도 갱신한다. 단 **investigation
   스냅샷은 코드 drift로 갱신하지 않고**(새 기록을 쓴다), 정정은 원래 계산 오류·오타에 한한다.
5. **목록 갱신 확인** — 신규 문서가 해당 `README.md` 표에 올라가 있나.

## 검토 후 자가 점검

- [ ] 본문 **+ 그림 캡션·몽타주 라벨**의 수치를 원자료에서 재계산했나?
- [ ] 곱·합·비율·배수를 직접 다시 계산했나? (어림식이 실측과 맞나)
- [ ] 코드 참조(함수·라인·`carver/` 경로 접두)를 grep으로 확인했나?
- [ ] 상호 참조·README·**asset 경로**의 대상이 실제 존재하나?
- [ ] "통과/무손실/동일" 주장의 검증을 실제로 실행했나? (investigation은 추론으로 약화)
- [ ] 단정한 것이 모두 측정·실행으로 뒷받침되나? 추론은 추론으로 표시됐나?
- [ ] 수치 출처·단위·**용어 표기**가 문서 전체에서 일관되나?
- [ ] 유형별 능동 검증(spec 모듈/drift, report 약어/민감정보, investigation 기각가설/스냅샷,
      ADR 대안/Superseded, reference 사실 대조)을 했나?
- [ ] 수정으로 바뀐 수치를 인용하는 다른 문서까지 전파했나?

## 자주 하는 실수

| 실수 | 수정 |
|------|------|
| 작성 자가점검만 믿고 재계산 생략 | 원자료에서 독립 재산출 |
| 본문 수치만 보고 그림 캡션·몽타주 라벨 누락 | 캡션·라벨 수치도 재계산 |
| 코드 참조를 눈으로만 / 경로 접두 누락 | grep으로 함수·라인·`carver/` 경로 대조 |
| 링크만 보고 이미지 asset 누락 | 링크·asset 경로 존재 확인 |
| 추론을 단정으로 서술 | 측정 안 한 것은 "추론" + 근거 |
| 측정 출처·용어 혼재 | 출처·단위·용어 표기 통일 |
| 주장만 하고 검증 미실행 | 테스트·빌드·diff를 실제로 실행·기록 |
| investigation 스냅샷에 사후 검증 삽입 | 미실행은 추론으로 약화, drift는 새 기록으로 |
| 유형별 점검을 자가점검 재독으로만 | 유형별 능동 행동(모듈 grep·약어·기각가설 확인) |
